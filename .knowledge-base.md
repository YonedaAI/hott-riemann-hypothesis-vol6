## 4. Six Paper Specs (Proof Sprint Order)

### Paper 01: FiniteBlaschkePacket

**Title:** Concrete Finite Blaschke Packet Model Spaces

**Abstract:** This paper defines finite Blaschke packets — finite collections
of complex points representing candidate off-critical zeta zeros — and
constructs the associated finite-dimensional model space `K_B^{(P)} = H^2 /
B_P H^2`. It proves that any nonempty packet yields a nonempty model space by
exhibiting an explicit reproducing kernel vector. This is the smallest
nonfake step toward `OffCriticalZeroDefectKernel` required by next-steps.md.

**Lean module:** `Vol6.FiniteBlaschkePacket`

**Principal theorem:**

```lean
theorem finite_packet_model_space_nonempty
    (P : FiniteBlaschkePacket)
    (h : Nonempty P.ZeroIndex) :
    Nonempty (finiteBlaschkeModelSpace P).carrier
```

**Prerequisites from vol5:**
- `Vol5.YonedaNymanTrackA.BurnolCore` — `HardySpaceH2`, `BurnolBlaschkeFactor`
- `Vol5.YonedaNymanTrackA.BlaschkeDefectObject` — `BlaschkeDefectObject` carrier shape
- `Vol5.V5aTrackAFormulation.TrackAFormulation` — `riemannZeta`

**Prerequisites from earlier vol6 papers:** none (first paper)

**Key proof obligations:**
1. Define `FiniteBlaschkePacket` and `FiniteBlaschkeModelSpace` as in next-steps.md §Immediate Next Lean File.
2. Define `finiteBlaschkeModelSpace : FiniteBlaschkePacket -> FiniteBlaschkeModelSpace` constructively: the carrier is `Fin P.ZeroIndex.card -> ℂ` (or equivalent finite quotient type); the kernel vector is the evaluation functional at any packet zero.
3. Prove `finite_packet_model_space_nonempty` by exhibiting the reproducing kernel `k_w(z) = (1 - B_P(z)\overline{B_P(w)}) / (1 - z\overline{w})` at any packet zero `w`.
4. Prove `kernelVector_nonzero` by nondegeneracy of the reproducing kernel on the model space (Gram matrix entry at coincident points is nonzero by basic Hardy space theory).
5. No Nyman density is needed. The argument is purely finite Hardy space theory.

### Paper 02: FiniteRankDetector

**Title:** Detector Completeness for Finite Blaschke Packets via Gram-Matrix Nondegeneracy

**Abstract:** For any finite Blaschke packet model space, this paper constructs

proof reduces to nondegeneracy of the Gram matrix of reproducing kernels at
the packet zero points, which is a standard Hardy/Pick matrix result. This
paper produces a concrete inhabitant of `FiniteModelSpaceDetector` for finite

### Paper 05: QuotientOrthogonalityAdmissibility

**Title:** Orthogonality by Cokernel Construction and Finite-Rank Admissibility

**Abstract:** This paper proves the remaining two components of
`BurnolBlaschkeRKHSDetectorSemantics`: quotient orthogonality/invisibility
and admissibility. Orthogonality follows by construction: `K_B = H^2 / B H^2`
is the cokernel of `B H^2 -> H^2`, so its elements are orthogonal to `B H^2`
in the Hardy inner product; this is an intrinsic property of the quotient map
and requires no Nyman density. Admissibility — the three atoms
`BlaschkeDefectDualizable`, `BlaschkeDefectPolarized`,
`BlaschkeDefectRegularized` — is proved first for finite packets via
finite-rank Hilbert space theory, then lifted to the canonical object under
regularized (not raw operator-norm) convergence.

**Key proof obligations:**

## 8. Open Questions and Risk Flags

### RF-01: opaque burnolBlaschkeFactor — CRITICAL

`burnolBlaschkeFactor : BurnolBlaschkeFactor` is declared as an `axiom` in
`BurnolCore.lean` line 42. Its field `modelSpaceCarrier : Type` is therefore
also not constructively defined. Paper 03 (OffCriticalDefectKernel) must
transport a vector from the finite Blaschke packet model space to
`burnolBlaschkeFactor.modelSpaceCarrier`. This transport is impossible without
either (a) replacing the `axiom burnolBlaschkeFactor` with a concrete
definition (making the model space carrier a concrete Lean type such as a
quotient or subtype of `ℂ → ℂ`), or (b) introducing a new `axiom`-free
identification between the finite-packet carrier and the axiom's type. Option
(a) requires modifying vol5 (forbidden). Option (b) requires building the
identification from the axiom's stated properties alone, which is impossible
without extending the axiom's API. Resolution: vol6 must define its own
concrete replacement for `burnolBlaschkeFactor` (call it
`concreteBurnolBlaschkeFactor`) that is definitionally equal or provably
equivalent, and then state the transport lemma between the vol6 concrete
carrier and vol5's opaque axiom. Agents must NOT add new axioms; they must make
the carrier concrete in vol6 and prove the equivalence using only the axiom's
existing type-level interface.

### RF-02: opaque DefectDetectedByRationalDilationRepresentable — CRITICAL

`DefectDetectedByRationalDilationRepresentable : BlaschkeDefectObject ->
DQPresheaf -> Prop` is declared `opaque` in `NoPhantomBlaschkeDefect.lean`
line 28. Paper 04 (externalization) and Paper 05 (orthogonality) both need to
prove instances of this predicate. Since it is opaque, no definitional unfolding
is possible; the only way to prove it is via lemmas that were also stated in
vol5. Vol6 agents must survey what lemmas, if any, vol5 provides for
constructing inhabitants of this opaque Prop. If none exist, vol6 must add
non-axiom-introducing API via transparent wrappers or must make a concrete
definition that is provably equivalent to the opaque one (using `propext` and a
characterization theorem from vol5 if such exists). Currently no such
characterization theorem is visible in the files surveyed.

### RF-03: opaque BlaschkeDefectDualizable / Polarized / Regularized — HIGH RISK

All three admissibility atoms are declared `opaque` in `NoPhantomBlaschkeDefect.lean`
lines 66–70. Paper 05 must prove concrete inhabitants of each. The same issue
as RF-02 applies: these are opaque Props with no currently-visible
characterization lemmas. Vol6 agents must either find existing vol5 lemmas that
construct these (none visible in current survey) or make them concrete by
providing constructors. The most likely legitimate route: define `concrete_BlaschkeDefectDualizable`, `concrete_BlaschkeDefectPolarized`, and `concrete_BlaschkeDefectRegularized` as transparent propositions in vol6 (without the `opaque` modifier), prove those, and use `propext` plus a bridge lemma to identify them with the opaque vol5 versions. This requires the vol5 opaque props to have been stated correctly, but does not require modifying vol5. Alternatively, add API to vol5 (which is forbidden); or add a new `axiom` (which is forbidden). This is the hardest open problem.

### RF-04: HardySpaceH2 and HardySubmodule are opaque axioms — HIGH RISK

Both `HardySpaceH2 : Type` and `HardySubmodule : Type` are declared via
`axiom` in `BurnolCore.lean` lines 19–21. There is no Mathlib-backed `H^2`
inner product available through these axioms. Papers 01–03 (which require
reproducing kernel computations and Gram matrix nondegeneracy) cannot use
Mathlib's Hardy space directly unless they work in an independent concrete type
and transport to the axiom type. Vol6 agents should use Mathlib's
`Hardy.H2` (if it exists) or construct a concrete surrogate for the finite
Blaschke packet case and prove the key lemmas there, then bridge to the opaque
axiom type at the transport step.
